◆ bound

LAZY_BOOLEAN bound ( const REAL &  x,
const int  k

friend

Multi-valued check whether 2^k bounds |x|.

If it is necessary to check that a number is small enough (e.g. given as the error of an approximating algorithm), bound(x,k) should be preferred over size(x)<k as there is no singularity at x=0.

The current approximation to \(x\) is deemed precise enough, predicate \(P_k(\hat x,x_\varepsilon)\), when \(\neg(\hat x<x_\varepsilon+2^{k-1}+2^e\wedge2^k<\hat x+x_\varepsilon)\) where \(\hat x=m\cdot2^e\). Otherwise, \(\bot\) is returned. These are the cases:

  1. \(\bot\): \(\neg P_k(\hat x,x_\varepsilon)\Longrightarrow\hat x+x_\varepsilon>2^k=2\cdot2^{k-1}>2(\hat x-x_\varepsilon)\Longrightarrow x_\varepsilon>\hat x/3>|x|/3\).
  2. \(T\): \(P_k(\hat x,x_\varepsilon)\wedge2^k\geq\hat x+x_\varepsilon>|x|\)
  3. \(F\): \(P_k(\hat x,x_\varepsilon)\wedge2^k<\hat x+x_\varepsilon \Longrightarrow\hat x\geq x_\varepsilon+2^{k-1}+2^e \Longrightarrow2^{k-1}\leq\check x-x_\varepsilon\leq|x|\).
Parameters
xreal number to bound the absolute value of
kbase-2 logarithm of the bound
Returns
\(\text{bound}(x,k)=\begin{cases} \bot,&\neg P_k(\hat x,x_\varepsilon)\Longrightarrow x_\varepsilon>|x|/3 \\ T,&P_k(\hat x,x_\varepsilon)\wedge2^k\geq\hat x+x_\varepsilon\Longrightarrow |x|<2^k \\ F,&P_k(\hat x,x_\varepsilon)\wedge2^k<\hat x+x_\varepsilon\Longrightarrow |x|\geq 2^{k-1} \end{cases}\)

According to documentation, this function should compute \(\begin{cases} T,&|x|\leq 2^k \\ F,&|x|\geq 2^{k-2} \\ \bot,&\text{otherwise} \end{cases}\) and does indeed compute a tightening.

◆ size

int size ( const REAL &  x)
friend

Returns a tight bound to the logarithmic value of |x|: \(2^{k-2}-2^e\leq|x|<2^k\), where e is x.vsize.exponent.

This is a multi-valued function.

For \(\hat x=m\cdot2^e\) = x.vsize and \(x_\varepsilon\) = x.error, this function computes \(k=\lceil\log_2(\hat x+x_\varepsilon)\rceil>\log_2|x|\) and via reiteration enforces that \(\hat x\geq x_\varepsilon+2^{k-2}+2^e\) which is equivalent to \(2^{k-2}\leq\check x-x_\varepsilon=(m-1)\cdot2^e-x_\varepsilon \leq|x|\), the guarrantee therefore is \(2^{k-2}\leq|x|<2^k\).

Parameters
xnon-zero real number
Returns
\(k\in\mathbb Z:2^{k-2}\leq|x|<2^k\), where min_exponent <= k < MP_max
Exceptions
iRRAM_Numerical_Exception(iRRAM_underflow_error)if x is exact with value zero. If x is not exact but zero, reiterations will be performed.
See also
iRRAM_REITERATE

    \[ \forall x,y\in[A;B]: |f(x)-f(y)| \leq K\cdot|x-y| = K\cdot|f(x)+f(y)|\cdot|f(x)-f(y)| \Longleftrightarrow 1/|f(x)+f(y)| \leq K \]

    Since \(\max_{x,y \in [A;B]} 1/|f(x)+f(y)| = 1/|2f(A)|\leq K\) for the function's argument \(x\in[x_c\pm x_\varepsilon]\) the error therefore is \(|f(x_c)-f(x)| \leq K|x-x_c| < Kx_\varepsilon \leq x_\varepsilon/(\sqrt{\check x}+\sqrt{\check x-x_\varepsilon}) < x_\varepsilon/(2\sqrt{\check x-x_\varepsilon})\).

    This case, \(x\geq A=\check x-x_\varepsilon>0\), is chosen when \(\check x\geq 4x_\varepsilon\Longrightarrow |x|\geq 3x_\varepsilon\), therefore \(|\sqrt{x_c}-\sqrt x| \leq x_\varepsilon/(2\sqrt{\check x-x_\varepsilon})\). The absolute precision used to compute \(\sqrt{x_c}\) is \(\max\{P,e[z]\}\) where \(z\geq x_\varepsilon/(2\sqrt{\check x-x_\varepsilon})\) and \(e[v]=e\) for any sizetype representation \(m\cdot2^e\) of \(v\). The relative precision is \(\max\{e[\check x]+P,e[z]\}\).

  • Otherwise, \(x=0\) or \(4x_\varepsilon\geq\check x=\hat x-2^e>\hat x\cdot(1-2^{-32})\), then \(\sqrt{|x|} <\sqrt{\hat x+x_\varepsilon} <\sqrt{4x_\varepsilon/(1-2^{-32})+x_\varepsilon} <2.25\sqrt{x_\varepsilon}\) and \(0\approx\sqrt{|x|}\) is chosen as an approximation with error \(\geq3\sqrt{x_\varepsilon}\).

Converts a base-10 string in scientific notation to REAL.

If \(p\) is the current working precision of iRRAM, \(k\) the number of digits in the integer part, \(z\) the number of leading zero-digits (including the fractional part) and \(g\) the exponent, and the represented value is not zero, then the result has a relative precision of roughly \((g+k-z)\log_2(10)-p\) bits, which corresponds to an absolute accuracy of \(2^p\).

Parameters
[in]spointer to base-10 string; format: WSDFE where
  • W = optional whitespace; it is discarded
  • S = optional sign
  • D = optional string of digits 0..9
  • F = optional: . followed by optional string of digits 0..9
  • E = optional: e or E followed by an optional sign followed by a string of digits 0..9
  • DF must be non-empty and not just contain .
[out]endptrwhen non-NULL, the pointer to the end of the represented number is stored in *endptr
Returns
a REAL object accurate to the current working precision.
Proof
The precision depends only on DF and E; let \(x_k\ldots x_{-n}=DF[1..|F|]\) and \(z\in\mathbb N:x[1..z]=\texttt 0^z\). To represent zero up to any accuracy, only 1 bit is required. So let \(x\neq0\) and \(b\cdot2^c\) with \(b\in[2^{-1};1)\) be the result. Then the error is

\[ (*)=|10^g\sum_{i=-n}^{k-z} x_i\cdot 10^i-2^cb| =|2^{\overbrace{(g+k-z+1)\log_2(10)}^{=:q}} \underbrace{\sum_{i=-n-(k-z+1)}^{-1}x_{i+k-z+1}\cdot10^i}_{=:x'\in[10^{-1};1)} -2^cb| \]

Now, MPFR's mpfr_strtofr() guarantees for the exact input \(2^qx'=:2^{\overline q}\overline b\) with \(\overline b\in[2^{-1};1)\) that the error is \(|2^{\overline q}\overline b-2^cb|<2^{\overline q-m}\), therefore

\[(*)=|2^qx'-2^{\overline q}\overline b+2^{\overline q}\overline b-2^cb| <|2^qx'-2^{\overline q}\overline b|+2^{\overline q-m}=2^{\overline q-m} \]

Since \(2^{\overline q}/2\leq 2^{\overline q}\overline b=2^qx'<2^q\), the error is bounded, \((*)<2^p\), when \(\overline q-p<m\leq q+1-p<\lceil(g+k-z+1)\log_2(10)\rceil+1-p\leq m'\). The exact precision is \(\max(10,m')\) where \(m'=1+\begin{cases} 1&z=k+n+1\\ \lfloor((g+k-z+1)\cdot 10+2)/3+1\rfloor-p&\text{otherwise} \end{cases}\)
